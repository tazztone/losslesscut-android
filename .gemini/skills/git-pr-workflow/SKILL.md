# Git PR Workflow
- **Conventional Commits**: Use `feat:`, `fix:`, `docs:`, `refactor:`.
- **Commit Style**: Focus on "why" in the commit message body.
- **PR Descriptions**: Briefly summarize changes, impact, and testing done.
